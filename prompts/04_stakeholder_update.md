# Prompt: Stakeholder Update

> Copy this prompt into Microsoft 365 Copilot (Teams, Word, or web).
> Paste your failure analysis from Lab Part 3 as the input context below the prompt.

---

Create a stakeholder-friendly test status update based on the failure analysis below.

**Audience:**
- Engineering manager
- Test lead
- Product owner

**Tone:**
- Clear
- Neutral
- Concise

**Include:**
- Current test status (pass/fail summary)
- Main failure pattern observed
- Release risk assessment
- What is known (confirmed facts)
- What is still uncertain (hypotheses, missing data)
- Recommended next actions with owners if possible
- Open questions or follow-up items

**Constraints:**
- Do not overstate the root cause — if it is uncertain, say so.
- Separate facts from assumptions.
- Do not invent data that is not in the failure analysis.
- Keep it suitable for a Teams message or email — no more than 300 words.
- Use bullet points for readability.
